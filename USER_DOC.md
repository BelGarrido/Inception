# User Documentation

## What This Stack Provides

This project runs a small web stack with three services:

- NGINX: the public entry point for the website and HTTPS traffic.
- WordPress with PHP-FPM: the application that renders the site.
- MariaDB: the database that stores WordPress content and users.

Traffic flows from the browser to NGINX, then to WordPress, and finally to MariaDB when data is needed.

## Start and Stop the Project

From the repository root, use the Makefile:

- `make all` builds the images, creates the local data directories, and starts the full stack.
- `make build` builds the images without starting them.
- `make up` starts the stack if the images already exist.
- `make stop` stops the running containers.
- `make clean` stops the stack and removes the containers and network.
- `make fclean` removes the containers, images, volumes, and the local data directory under `/home/anagarri/data`.

## Access the Website and Admin Panel

After the stack is running, open the website with:

- `https://anagarri.42.fr`

WordPress administration is available at:

- `https://anagarri.42.fr/wp-admin`

The browser must trust the self-signed certificate generated by the NGINX container.

## Locate and Manage Credentials

The project uses environment variables stored in `srcs/.env`. Create that file from `srcs/template.env` and fill in the required values before starting the stack.

Keep `srcs/.env` private. It contains the MariaDB credentials and the WordPress administrator and user credentials.

Important values include:

- MariaDB database name, user, password, and root password.
- WordPress database connection values.
- WordPress site title.
- WordPress admin account credentials.
- WordPress regular user credentials.

If you need to change credentials, update `srcs/.env` and restart the stack.

## Check That Services Are Running Correctly

You can verify the stack in a few simple ways:

- Open `https://anagarri.42.fr` and confirm the WordPress home page loads.
- Open `https://anagarri.42.fr/wp-admin` and confirm the login page appears.
- Run `make` targets such as `make stop` and `make up` to confirm the services stop and start cleanly.
- Use Docker commands such as `docker compose ps` to check that NGINX, WordPress, and MariaDB are running.

If the site does not load, check that the domain points to `127.0.0.1` in `/etc/hosts` and that the containers are running.
